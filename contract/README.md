# Contract — DOGL notation and architecture

This folder holds the **formal specification** of DOGL: notation rules and architecture. It is the stable contract for the community, tools, and AI.

**Contents:**

- **[tokens.md](tokens.md)** — Token list and rules: lexical form, context, and what may follow for each token (keywords, structure, elements, flows, expressions, identifiers, optional inline bounds, comments).
- **Notation** (to be added) — Full syntactic rules for `.dogl`.
- **Architecture** (to be added) — Layers (Domain, Application, Adapters), data flow, and boundaries.

Use this as the single source of truth when implementing parsers, editors, or when asking AI to reason about DOGL.
